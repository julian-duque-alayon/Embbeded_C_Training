# 📡 STM32 Development Hub

Welcome to the heart of the STM32 section. This directory contains the templates, tools, and projects used to master ARM Cortex-M microcontrollers from the ground up.

---

## 🚀 The Philosophy: Beyond the IDE
The goal of this repository is to break free from the "black box" of heavy IDEs like STM32CubeIDE or STM32 Studio. We aim for:
*   **Minimalism**: Writing C code in **VS Code** for a lightweight and lightning-fast experience.
*   **Transparency**: Understanding every line of code, from the startup script to the linker file.
*   **Low-Level Mastery**: Moving away from high-level abstractions to talk directly to the hardware via registers and pointers.

---

## 🏛️ Origins: The "Template" Story
Every project here starts from a solid foundation. Our standard structure wasn't created by hand from scratch—it was carefully transitioned:

1.  **The Seed (STM32CubeMX)**: We used the standalone STM32CubeMX tool to generate the initial hardware configuration (IOs, Clocks, Peripherals). This is the only "crutch" we use, as it avoids manually mapping thousands of register addresses.
2.  **The Extraction**: We took the raw code generated by CubeMX and stripped away the IDE-specific clutter.
3.  **The Transformation**: We integrated a custom **CMake** build system and **VS Code** tasks to turn it into a portable, professional-grade development environment.
4.  **The Result**: The `Template_NucleoF413ZH` folder—a clean, modular blueprint ready for any custom application.

## 🛠️ Hardware Workflow (WSL Support)

All STM32 boards in this repository use **ST-LINK** for debugging and flashing. Since hardware is connected via USB to Windows but we use WSL for development, follow these steps:

### 1️⃣ Windows Side (PowerShell)
Expose the USB debugger to WSL:
```powershell
usbipd list # Identify the BUSID
usbipd attach --wsl --busid <YOUR_BUSID> --auto-attach
```

### 2️⃣ WSL Side (Linux Terminal)
Verify the connection:
```bash
lsusb # Look for "STMicroelectronics ST-LINK"
```

## 💡 Quick Troubleshooting
*   **Connection Lost**: Check `lsusb`. You might need to re-run `usbipd attach` in PowerShell.
*   **Permission Denied**: Ensure your user is in the `plugdev` group or use `sudo` for OpenOCD.
